# Agent guidelines

## Git workflow

- NEVER commit or push directly to `main`.
- For any change, create a feature branch (e.g. `feature/<short-name>`), commit there, and push that branch.
- Open a pull request against `main` for review. The user merges PRs; do not merge to `main` yourself.
- Only operate on `main` directly if the user explicitly asks for it in that request.
